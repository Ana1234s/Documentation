00003 # PocketMine-MP Contribution Guidelines
00004 
00005 Before contributing to PocketMine-MP, please read this. Also, take a look if your contribution fits the PocketMine-MP goals below.
00006 
00007 
00008 ## I've a question
00009 * For questions, please refer to the _#pocketmine_ or _#mcpedevs_ IRC channel on Freenode. There is a [WebIRC](http://webchat.freenode.net?channels=pocketmine,mcpedevs&uio=d4) if you want.
00010 * You can ask directly to _[@PocketMine](https://twitter.com/PocketMine)_ in Twitter, but don't expect an inmediate reply.
00011 
00012 ## I want to create an issue
00013 * First, use the [Issue Search](https://github.com/PocketMine/PocketMine-MP/search?ref=cmdform&type=Issues) to check if anyone has reported it.
00014 * Is your issue related to a Plugin? If so, please contact their original author instead of reporting it here.
00015  * And no, we won't update a Plugin because you need it.
00016 * When reporting, give as much info as you can, and if the Issue is a crash, give the Crash Dump.
00017 * Issues should be written in English.
00018 
00019 ## I want a new feature / I want to contribute code
00020 * Use the [Pull Request](https://github.com/PocketMine/PocketMine-MP/pull/new) system, your request will be checked and discussed.
00021 * __Create a single branch for that pull request__
00022 * If you want to be part of PocketMine-MP, we will ask you to.
00023 * Code using the syntax as in PocketMine-MP. See below for an example.
00024 * The code must be clear and written in English, comments included.
00025 
00026 
00027 __Thanks for contributing to PocketMine-MP!__

00031 ### Code syntax
00032 
00033 It is mainly [PSR-2](https://github.com/php-fig/fig-standards/blob/master/accepted/PSR-1-basic-coding-standard.md) with a few exceptions.
00034 * Opening braces MUST go on the same line.
00035 * `else if` MUST be written as `elseif`. _(It is in PSR-2, but using a SHOULD)_
00036 * Control structure keywords or opening braces MUST NOT have one space after them.
00037 * Code MUST use tabs for indenting.
00038 * Long arrays MAY be split across multiple lines, where each subsequent line is indented once. 
00039 * Files MUST use only the `<?php` tag.
00040 * Files MUST NOT have an ending `?>` tag.
00041 * Code MUST use namespaces.
00042 * Strings SHOULD use the double quote `"` except when the single quote is required.
00043 * Argument lists MAY NOT be split across multiple lines, except long arrays.
